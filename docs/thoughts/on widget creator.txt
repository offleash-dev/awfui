 keep AFUI lean, keep the core pure, and let a helper script generate the boilerplate when someone wants it.  

A compile‑time helper script that generates C++
This is the cleanest, safest, most embedded‑friendly option.

You write something like:

Code
Dialog "Warning" id:WARN size:(screenWidth - 10) {
    Label "Something happened!" id:WHAT
    Button "OK" id:BtOK
}
A Python script (or CMake custom command) turns that into:

cpp
void buildWarningDialog(AFScreen* s) {
    auto* dlg = new AFDialog(…, WARN);
    dlg->addWidget(new AFLabel(…, "Something happened!", WHAT));
    dlg->addWidget(new AFButton(…, "OK", BtOK));
    s->addWidget(dlg);
}
Pros:

zero runtime cost

no parser on device

no memory overhead

no dynamic allocation unless you choose it

easy to debug (you can inspect the generated C++)

easy to extend

This is the “Qt .ui file” model, but tiny and embedded‑friendly.

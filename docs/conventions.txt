Coding conventions:

Mantra: Clarity.  Not extra junk.  Think embedded.

WRT containers:
No STL (use etl)
https://www.etlcpp.com/

WRT error handling:
No exceptions.  
Should return something checkable if an error is possible 
    bool success is fine
    null is fine
    an enum defined in AFResult.h is fine
    asserts are for programmer errors

WRT namespaces:
no namespaces.   use AF* prefix

WRT member variables:
m_* for member variables
s_* for static variables (we don't have any yet)

k* for contants (not all caps)

WRT virtual methods:
virtual methods should use override

WRT factories:
factories when appropriate (e.g. screen factory in AFWorld)
